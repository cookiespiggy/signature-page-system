# 环境配置

> 签字页管理系统的开发环境约束和跨平台问题。

## 环境总览

本项目使用 **OrbStack VM（oh-agent）** 隔离后端基础设施，前端的浏览器/Node 工具链在宿主机运行。

```
┌─────────────────────────────────────────────────────┐
│                  macOS 宿主机 (本机)                    │
│  ┌────────────────────────────────────────────────┐  │
│  │             oh-agent VM (Ubuntu)                │  │
│  │  ┌──────────┐ ┌──────────┐ ┌───────────────┐  │  │
│  │  │ FastAPI  │ │  SQLite  │ │ OpenCode Svr  │  │  │
│  │  │ :8000    │ │ .db file │ │ :4096         │  │  │
│  │  └──────────┘ └──────────┘ └───────────────┘  │  │
│  └────────────────────────────────────────────────┘  │
│  ┌──────────────────────────────────────┐            │
│  │  Vite Dev Server (:5173)              │            │
│  │  → proxy /api → VM :8000              │            │
│  └──────────────────────────────────────┘            │
└─────────────────────────────────────────────────────┘
```

## 开发场景矩阵

| 场景 | 在哪执行 | 原因 |
|------|---------|------|
| **后端** `uv run uvicorn` | `orb -m oh-agent` | .venv 是 Linux Python，宿主机不可用 |
| **前端** `npm run dev` | 宿主机直接跑 | 浏览器在 macOS，Vite 无平台限制 |
| **OpenCode Server** `opencode serve` | `orb -m oh-agent` | API 走 VM 内 HTTP |
| **数据库迁移** `alembic upgrade head` | `orb -m oh-agent` | 需要 .venv |
| **pytest** | `orb -m oh-agent` | 需要 .venv |
| **AI 场景端到端测试** (调 OpenCode) | `orb -m oh-agent .venv/bin/python` | 需要 .venv + 访问 OpenCode :4096 |
| **AI 场景测试** (Mock 模式) | 同上即可 | Mock 无外部依赖 |
| **单次脚本**（如调试 LLM 响应） | `orb -m oh-agent .venv/bin/python3` | 需要 httpx + .venv |
| **模板 DOCX 操作**（python-docx） | `orb -m oh-agent` | 依赖 .venv |
| **查看/编辑 DB** `sqlite3` | 宿主机或 `orb -m oh-agent` | SQLite .db 在共享存储上 |
| **Git 操作** | 宿主机直接跑 | Git 没有平台限制 |
| **前端联调**（Vite → proxy → VM :8000） | 前端宿主机 + 后端 VM | 各自最自然的环境 |

### 后端 LLM Provider 选择场景

| LLM_PROVIDER | 启动方式 | 适用场景 |
|-------------|---------|---------|
| `opencode`（默认） | 先 `orb -m oh-agent opencode serve`，再 `orb -m oh-agent uv run ...` | 日常开发与 AI 功能验证，需 OpenCode Server 运行 |
| `mock` | `orb -m oh-agent LLM_PROVIDER=mock uv run ...` | 离线开发调试，不依赖 OpenCode Server |
| `openai` | 需设置 `OPENAI_API_KEY` | 生产环境或需要 GPT-4o 等高级模型 |

## .venv 跨平台陷阱

`.venv/` 在 OrbStack 共享文件系统上，但 **Python 二进制文件不跨平台**：

| 在哪创建 | 在哪能用 | 现象 |
|---------|---------|------|
| macOS 宿主机 | 仅 macOS | `uv run` 正常 |
| oh-agent VM（Linux） | 仅 Linux VM | `orb -m oh-agent uv run` 正常 |
| macOS 创建后到 VM 用 | ❌ | `ModuleNotFoundError: No module named 'python'` |

**补救：在 VM 内重建**
```bash
orb -m oh-agent rm -rf /Users/jimmy/Workspaces/junhe-mvp/backend/.venv
orb -m oh-agent uv sync
```

**关于 `orb -m oh-agent uv run` 的原理**：
- `orb -m oh-agent` 相当于在 VM 内执行后续命令
- VM 内的 `uv` 使用共享文件系统上的 `.venv/`
- 因为 .venv 是 Linux Python，所以只能在 VM 内执行
- 宿主机如果也需要跑后端，需要在 macOS 下单独 `uv sync`

## 后端启动

所有后端命令在 oh-agent VM 内执行：

```bash
# 默认 OpenCode LLM（先确保 opencode serve 在运行）
orb -m oh-agent uv run uvicorn app.main:app --port 8000

# 切回 Mock（离线环境）
orb -m oh-agent LLM_PROVIDER=mock uv run uvicorn app.main:app --port 8000
```

## 数据库迁移

```bash
orb -m oh-agent uv run alembic upgrade head
orb -m oh-agent uv run alembic revision --autogenerate -m "描述"
```

## 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DATABASE_URL` | `sqlite:///./data/junhe.db` | 数据库连接 |
| `LLM_PROVIDER` | `opencode` | LLM 提供方：`opencode`/`mock`/`openai` |
| `GENERATED_DIR` | `data/generated` | 生成文件存储目录 |

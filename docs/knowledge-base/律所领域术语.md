# 律所签字页领域术语

> 本文档沉淀签字页管理系统中涉及的律所业务概念，帮助理解变量命名、分类逻辑和业务背景。

---

## 一、核心概念

| 术语 | 英文 | 解释 |
|------|------|------|
| 签字页 | Signing Page | 法律意见书/决议文件末尾的签署页，各方签章处。产品核心产出物 |
| 法律意见书 | Legal Opinion | 律所就某项交易出具的正式法律意见文件，签字页附于其后 |
| 股东大会决议 | Shareholder Resolution | 公司股东（自然人或机构）就重大事项表决形成的决议文件 |
| IPO | Initial Public Offering | 首次公开发行股票，律所最常见的业务场景之一 |
| 尽职调查 | Due Diligence | 律所对目标公司进行的法律合规审查，签字页往往是尽调报告的组成部分 |

---

## 二、角色与主体

| 术语 | 英文 | 业务含义 |
|------|------|---------|
| 目标公司 | Target Company | 被尽调/上市/交易的主体，如"海通证券股份有限公司" |
| 律师事务所 | Law Firm | 出具法律意见书的律所，如"君合律师事务所上海分所" |
| 律师事务所负责人 | Law Firm Director | 律所主任/负责人，代表律所在签字页上签章 |
| 经办律师 | Handling Lawyer | 具体负责该项目的签字律师，**可多值**（通常 2 人） |
| 自然人股东 | Natural Shareholder | 以个人身份持有公司股份的股东 |
| 机构股东 | Institutional Shareholder | 以法人实体持有公司股份的股东 |
| 授权代表 | Authorized Representative | 机构股东委派在决议上签字的人，如"执行事务合伙人委派代表" |
| 交易所 | Exchange | 上市交易场所，如"上海证券交易所"、"深圳证券交易所"、"北交所" |

---

## 三、文档类型与场景

| 术语 | 业务含义 |
|------|---------|
| 法律意见书 | IPO 项目中律所出具的核心文件，律师对发行人合法性发表专业意见 |
| 招股说明书 | IPO 项目向公众披露的招募文件，签字页由全体董事签署 |
| 股东大会决议 | 股东对重大事项（如增资、重组、分红）的决议文件 |
| 投资者类型 | 区分"普通投资者"还是"合格投资者"，影响签字格式和披露要求 |
| 文件类型 | 区分不同文书类型，可能影响签字页的格式版式 |

## 四、签字页变量详解

### 4.1 通用变量（跨模板共享）

| Key | 中文名 | 数据类型 | 为什么是必备 |
|-----|--------|---------|------------|
| `target_company_name` | 目标公司名称 | `company_name` | 签字主体是谁，所有签字页都需要 |
| `signing_date` | 签署日期 | `date` | 法律文件生效的时间锚点 |

### 4.2 律所签字页

| Key | 中文名 | 业务用途 |
|-----|--------|---------|
| `law_firm_name` | 律师事务所名称 | 明确出具意见的律所主体，如"君合律师事务所" |
| `law_firm_director` | 律师事务所负责人 | 律所主任签章处的对应人名 |
| `handling_lawyer` | 经办律师 | **多值变量**；IPO 法律意见书通常由 2 名签字律师出具 |
| `exchange_name` | 交易所名称 | 上市地，如"上海证券交易所" |
| `document_type` | 文件类型 | 区分不同意见书类型（如补充法律意见书 vs 正式法律意见书） |
| `target_investor_type` | 投资者类型 | 向特定对象发行 vs 公开发行，影响合规要求 |

### 4.3 自然人股东签字页

| Key | 中文名 | 业务用途 |
|-----|--------|---------|
| `natural_shareholder_name` | 自然人股东姓名 | 签署人的身份标识 |
| `natural_shareholder_id_number` | 自然人股东身份证号 | 身份验证，格式校验 `^\d{17}[\dXx]$` |
| `meeting_year` | 股东大会年份 | 如"2026"，决议文件的年份标识 |
| `meeting_session` | 股东大会次数 | 如"第一次"、"第二次" |

### 4.4 机构股东签字页

| Key | 中文名 | 业务用途 |
|-----|--------|---------|
| `institutional_shareholder_name` | 机构股东名称 | 法人主体名称，如"XX投资管理有限公司" |
| `authorized_representative_name` | 授权代表姓名 | 代表该机构签字的人 |

## 五、变量分类体系

系统将变量归入以下分类，用于前端分组展示：

| 分类 | 包含的变量 | 业务含义 |
|------|-----------|---------|
| `company` | `target_company_name`, `exchange_name`, `institutional_shareholder_name` | 公司/机构相关的主体信息 |
| `lawyer` | `law_firm_name`, `law_firm_director`, `handling_lawyer` | 律所及律师角色 |
| `shareholder` | `natural_shareholder_name`, `natural_shareholder_id_number`, `authorized_representative_name` | 股东角色信息 |
| `meeting` | `meeting_year`, `meeting_session` | 会议相关信息 |
| `document` | `document_type`, `target_investor_type` | 文件元信息 |
| `date` | `signing_date` | 时间类信息 |

## 六、命名规约

### 变量 key 设计原则

```
{业务实体}_{属性}

示例：
target_company  → target_company_name
natural_shareholder → natural_shareholder_name
```

- 使用全小写 **snake_case**
- 属性后缀惯例：`_name` 名称、`_date` 日期、`_id_number` 证件号、`_type` 类型
- 多值变量（`is_multiple=True`）在数据库存为多行 `key_1`, `key_2`, ...，如 `handling_lawyer_1`, `handling_lawyer_2`

## 七、常见业务知识问答

**Q: 为什么经办律师通常是 2 人？**  
A: 依据《律师事务所从事证券法律业务管理办法》，IPO 法律意见书必须由 2 名执业律师签字出具。

**Q: 什么是"合格投资者"？**  
A: 符合《证券期货投资者适当性管理办法》认定标准的投资者，与"普通投资者"相对，影响签字页的声明条款。

**Q: 机构股东和自然人股东的区别是什么？**  
A: 自然人股东以个人身份签署，需提供身份证号；机构股东以法人身份签署，需注明授权代表。两套签字页模板不同，不可混用。
